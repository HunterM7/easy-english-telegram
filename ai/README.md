# Настройка AI-команды

## Цель

Сделать процесс, где заказчик управляет задачами, а агенты выполняют работу через оркестратора.

## Состав файлов

- `ai/workflow.md` - жизненный цикл задачи, DoR, DoD, эскалации
- `ai/agents/*.md` - роли, зоны ответственности и ограничения
- `ai/templates/*.md` - шаблоны брифа, ТЗ, UI-kit-спецификации, дизайн-handoff и плана тестирования
- `.cursor/rules/*.mdc` - постоянные правила поведения в Cursor

## Как работать (кратко)

1. Заказчик дает короткий запрос.
2. Оркестратор задает уточняющие вопросы и заполняет `task-brief.md` (бриф задачи).
3. Для UI-задач дизайнер готовит `ui-kit-spec.md`, затем `design-handoff.md` и материалы в Figma.
4. Оркестратор формирует `tech-spec.md` (ТЗ) и назначения.
5. Фронтенд и бэкенд реализуют и дают подтверждение.
6. Ревьюер проверяет код и возвращает замечания.
7. QA проверяет по `test-plan.md`.
8. Оркестратор публикует финальный статус и готовность.

## Правила для задач "сделать по референсу"

- В брифе обязательно фиксировать ссылку на референс и `node-id`.
- До начала правок дизайнер возвращает Design Contract:
  - какие блоки войдут в экран,
  - какие элементы не будут добавляться.
- Если заказчик пишет "без лишнего", запрещено расширять контентный состав экрана.
- На итерациях править только подтвержденные расхождения (цвет, шрифт, структура, отступы), без смены scope.
