# Пилотная проверка агента-дизайнера (приоритет на сегодня)

## Цель пилота

Проверить, что агент-дизайнер умеет на основе одного референс-экрана:

1. собрать UI-kit и правила дизайн-системы;
2. подготовить дизайн-handoff для фронтенда;
3. передать артефакты без потери требований.

## Вход

- Один существующий экран в Figma (ссылка)
- Короткий запрос заказчика (2-3 предложения)
- PRD проекта

## Шаги пилота

1. Оркестратор проводит короткое интервью и фиксирует бриф.
2. Дизайнер заполняет `ui-kit-spec.md` по референс-экрану.
3. Дизайнер в Figma формирует UI-kit (токены, компоненты, состояния).
4. Дизайнер проектирует 1-2 новых экрана на базе UI-kit.
5. Дизайнер заполняет `design-handoff.md` и передает ссылки на фреймы.
6. Фронтенд делает быструю оценку реализуемости (без полной разработки).
7. Оркестратор фиксирует итоги пилота и решение по улучшениям.

## Критерии успеха

- [ ] В `ui-kit-spec.md` есть токены, компоненты и состояния
- [ ] В `design-handoff.md` перечислены экраны и состояния
- [ ] Фронтенду понятны правила реализации без дополнительных расшифровок
- [ ] Нет критичных расхождений с PRD
- [ ] Время на подготовку дизайна заметно меньше ручного процесса
- [ ] Перед стартом зафиксирован Design Contract (референс, обязательные блоки, что не добавляем)
- [ ] Финальный экран не содержит лишних элементов относительно референса
- [ ] Тема (light/dark), базовая палитра и типографическая иерархия совпадают с референсом
- [ ] Для каждой итерации зафиксирован короткий diff изменений

## Что фиксируем по итогам

- Что сработало хорошо
- Где агент ошибался или недоговаривал
- Какие правила нужно усилить в `ai/agents/designer.md`
- Какие пункты добавить в `ui-kit-spec.md` и `design-handoff.md`
