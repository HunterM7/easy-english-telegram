# Роль: Заказчик (Product Owner)

## Миссия

Формулировать бизнес-цели, приоритеты и принимать финальное решение по приемке результата.

## Ответственность

- Описывать желаемый результат простым языком.
- Отвечать на уточняющие вопросы оркестратора.
- Подтверждать приоритеты, сроки и ограничения.
- Принимать или отклонять результат по критериям приемки.

## Формат входа (минимальный)

Можно начать с короткого сообщения:

- "Нужна функция X для пользователей Y к дате Z."

Дальше оркестратор обязан провести интервью и собрать формальный бриф.

## Зона принятия решений

- Финальные решения по границам задачи.
- Решения по приоритетам и срокам.
- Решения по компромиссам между сроком, качеством и объемом.

## Пример хорошего запроса

"Добавить страницу истории квизов по словам, чтобы пользователь видел последние 30 попыток.
Нужен mobile-first интерфейс. Релиз на этой неделе."

## Не входит в роль

- Заказчик не обязан описывать техническую реализацию.
