# Skill: api-contract-guard

## Цель
Контролировать изменения API-контрактов и предотвращать неожиданные регрессии клиентов.

## Использовать когда
- Меняются DTO, response schema, названия/типы полей, коды ошибок, обязательность полей.

## Шаги
1. Определить тип изменения: backward-compatible или breaking.
2. Перечислить затронутые клиенты/экраны.
3. Обновить документацию API.
4. Проверить fallback/миграционный путь для breaking-изменений.

## Формат результата
- Тип изменения
- Затронутые контракты
- Миграционный план (если нужен)
- Подтверждение тестами
