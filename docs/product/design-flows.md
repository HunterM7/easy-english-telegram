# Карта экранов и пользовательский флоу

[Docs](../README.md) → [Product](README.md) → **design-flows**

Документ для дизайнеров: план всех экранов и способов перехода между ними. Исходный продуктовый контекст — [PRD](PRD.md).

**Описание каждого экрана по отдельности** (содержимое, навигация, состояния) — в [Client: Screens](../client/screens/README.md).

Цель — дать однозначную карту экранов и флоу. После обсуждения структуры раздел можно дополнять и уточнять.

**Важно:** на первом этапе **основное приложение** (главная, темы, тренажёр) доступно **только в Telegram Mini App**. Открытие сайта в браузере показывает **только посадочную** с переходом к боту; сценарий «войти в веб-приложение из браузера» — в планах ([API: авторизация](../server/api/auth.md)).

## 1. Схемы пользовательского флоу

### Схема 1 — Точки входа и главные экраны

```
  Mini App (Telegram)              Браузер (посадочная)
            │                                │
            │                         описание + CTA
            │                         «Открыть в Telegram»
            │                                │
            │                         → бот → Mini App
            │                                │
            └────────────┬───────────────────┘
                         ↓
                ГЛАВНАЯ (маскот)
                         │
           ┌─────────────┼─────────────┐
           ↓             ↓             ↓
     Список тем      Профиль    (нижнее меню:
           │             │      переключение между ними)
           ↓
     Содержимое темы
     (Теория + список уроков)
           │
      ┌────┴────┐
      ↓         ↓
   Теория   Тренажёр
      │         │
      └── Назад ┘
```

*Браузер без Telegram:* только посадочная; полноценный флоу приложения — после перехода к боту и открытия Mini App.

### Схема 2 — Навигация внутри темы

```
  Список тем
       │
       ↓
  Содержимое темы
  (Теория + уроки)
       │
   ┌───┴───┐
   ↓       ↓
Теория  Тренажёр
   │       │
   └─ Назад ┘
```

### Схема 3 — Флоу в тренажёре

```
  Задание
     │
     ↓
  Ответ верный?
     │
   ┌─┴─┐
   ↓   ↓
  Да   Нет
   │   │
   │   ↓
   │   Показать правильный ответ
   │   │
   │   ↓
   │   [Продолжить] ──→ снова Задание
   │
   ↓
  Успех + озвучка
   │
   ↓
  Есть ещё задания?
   │
   ├── Да ──→ Задание
   │
   └── Нет
        │
        ↓
     Урок завершён
        │
   ┌────┴────┐
   ↓         ↓
Свободный  Завершить
  режим      урок
   │         │
   ↓         ↓
Задание  Содержимое темы
```

### Таблица переходов (для справки)

| Откуда | Куда | Действие |
|--------|------|----------|
| Браузер | Посадочная | Открытие по ссылке на сайт |
| Посадочная | (вне приложения) | Кнопка «Открыть в Telegram» → бот → Mini App |
| Telegram | Главная | Старт Mini App |
| Главная | Список тем, Профиль | Нижнее меню |
| Список тем | Содержимое темы | Выбор темы |
| Содержимое темы | Теория или Тренажёр | Выбор пункта |
| Тренажёр | След. задание / Урок завершён | Правильный ответ |
| Тренажёр | То же задание (продолжить) | Ошибка → Продолжить |
| Урок завершён | Тренажёр (свободный режим) или Содержимое темы | Выбор кнопки |
| Любой экран | Предыдущий | Кнопка «Назад» |

## 2. Точки входа

- **Вход из Telegram (Mini App):** пользователь сразу попадает на **Главную** (экран с маскотом). Авторизация по `initData` не оформлена отдельным экраном.
- **Вход по ссылке в браузере:** **Посадочная страница** с переходом к боту; обучение — после открытия Mini App из Telegram.

## 3. Полный список экранов

| № | Экраны | Краткое описание | Как попасть на экран |
|---|--------|------------------|----------------------|
| 1 | Посадочная страница | Описание приложения, CTA «Открыть в Telegram» (бот) | Только при открытии сайта в браузере |
| 2 | Главная | Маскот (попугайчик), настроение, баланс зернышек; при необходимости приветствие, ссылки на «Мой попугай» / настройки | Старт Mini App в Telegram; кнопка «Главная» в нижнем меню (если есть) |
| 3 | Список тем | Список тем (Простое время, Герундий, Пассивный залог и т.д.) | Кнопка «Уроки» / «Темы» в нижнем меню |
| 4 | Список содержимого темы | Пункт «Теория» + список уроков выбранной темы | Клик по теме в списке тем |
| 5 | Теория темы | Текст и таблицы: грамматика, объяснение темы | Клик по пункту «Теория» в списке содержимого темы |
| 6 | Тренажёр (страница урока) | Задание: подсказка, варианты слов/ответов, кнопки «Подсказка», «Удалить последнее слово»; футер с прогрессом в % | Клик по уроку в списке содержимого темы |
| 7 | Состояние после правильного ответа (в тренажёре) | Сообщение об успехе, озвучка (если включено автопрослушивание), кнопка «Прослушать ещё раз»; настройка автопрослушивания (если выведена здесь); переход к следующему заданию | Правильный ответ в тренажёре |
| 8 | Состояние после ошибки (в тренажёре) | Сообщение об ошибке, правильный ответ, кнопка «Продолжить» / «Следующее задание» | Неправильный ответ в тренажёре |
| 9 | Экран успешного завершения урока | Сообщение о завершении урока; кнопки «Тренировка в свободном режиме» и «Завершить урок» | Выполнено последнее задание урока при первом прохождении |
| 10 | Настройки (профиль) | Профиль, кнопка «Достижения» (с баджем), статистика, настройки (тема, автопрослушивание) | Пункт «Настройки» в нижнем меню |
| 11 | Достижения | Список всех достижений: не получено (серые), готово к активации (яркие, кликабельны — забрать награду), получено (в цвете + дата) | Кнопка «Достижения» в настройках |
| 12 | Страница ошибки (404) | Сообщение об ошибке, переход назад или на главную | Неверный URL или несуществующий ресурс |

## 4. Навигация и переходы (кратко)

- **Главная** ↔ **Список тем** — через нижнее меню.
- **Список тем** → **Список содержимого темы** — по выбору темы; «Назад» → Список тем.
- **Список содержимого темы** → **Теория** или **Тренажёр** — по выбору пункта; «Назад» → Список содержимого темы.
- **Тренажёр** → следующее задание или экран **Успешное завершение урока** — по результату прохождения заданий; «Назад» → Список содержимого темы.
- **Успешное завершение урока** → снова тренажёр (свободный режим) или **Список содержимого темы** — по кнопке.
- **Главная** / нижнее меню → **Информация о пользователе**; «Назад» → предыдущий экран или главная.

## 5. Что уточнить при обсуждении с дизайнерами

- Точный состав нижнего меню (иконки, подписи): Главная, Уроки/Темы, Профиль?
- Где именно выводить настройку автопрослушивания: только в профиле, только после правильного ответа или в обоих местах.
- Состояния загрузки и ошибок сети на каждом экране (скелетоны при загрузке контента, спиннер в кнопке при отправке; кнопка «Обновить» только при ошибке загрузки).
- Достижения: полный список, визуал карточек (иконка, название, описание, награда, дата получения), примеры — см. PRD раздел 3.5 и п. 14.
- Экраны оплаты подписки: где в флоу (блокировка урока, отдельный экран «Оформить подписку» и т.д.).
- Посадочная: точный текст CTA и ссылка на бота; при появлении браузерной авторизации — второй CTA (см. PRD и [планы API](../server/api/auth.md#планы-браузерная-сессия-и-выход)).
