# API: Тренажёр (урок и упражнения)

[Docs](../../README.md) → [Server](../README.md) → [API](README.md) → **Тренажёр**

Эндпоинты для получения следующего упражнения урока и отправки ответа. Логика: клиент запрашивает **одно** упражнение за раз; после успешной проверки ответа запрашивает следующее. Сервер сам выбирает, какое упражнение вернуть (см. [PRD, раздел 3.4.1](../../product/PRD.md)). Все роуты защищённые — в заголовке передаётся **`accessToken`**: `Authorization: Bearer <accessToken>`.

---

## GET /v1/lessons/:lessonId/exercises/next

Возвращает одно упражнение для урока. Вызывается при открытии урока и после каждого успешно выполненного упражнения. Сервер выбирает по правилам: пока пользователь не прорешал все упражнения урока — только ещё не выполненные; когда все выполнены хотя бы раз — произвольный порядок с исключением последних 5 выполненных в этом уроке.

**Заголовки запроса**

| Заголовок        | Значение |
|------------------|----------|
| `Authorization`  | `Bearer <accessToken>` |

**Параметры пути**

| Параметр    | Тип   | Описание |
|-------------|--------|----------|
| `lessonId`  | string | Идентификатор урока (сущность **Lesson** в API). |

**Успешный ответ** — `200 OK`

Тело — один объект упражнения (поле `exercise` в JSON или корневой объект — по договорённости контракта):

| Поле         | Тип   | Описание |
|--------------|--------|----------|
| `id`         | string | Идентификатор упражнения. |
| `type`       | string | Тип: `sentence_builder`, `multiple_choice` и т.д. |
| `question`   | string | Вопрос/подсказка (например, текст на русском для сбора предложения). |
| `options`    | array  | (для multiple_choice) Варианты ответа. |
| `words`      | array  | (для sentence_builder) Слова для выбора (включая лишние). |
| …            | …      | Остальные поля по типу (см. контент и [implementation](../implementation.md)). |

Если упражнений в уроке нет или не осталось подходящих по правилам — на усмотрение контракта: `200` с пустым объектом / `null` или `404`. Рекомендуется `200` с полем `exercise: null` и полем `lessonProgress` (прогресс урока в %, флаг «все пройдены» и т.п.), чтобы клиент мог показать экран «Урок завершён».

**Ошибки**

| Код | Описание |
|-----|----------|
| 401 | Токен отсутствует или невалиден. |
| 403 | Нет доступа к уроку (например, урок в платной теме, подписка не активна). |
| 404 | Урок не найден. |
| 500 | Внутренняя ошибка сервера. |

## POST /v1/lessons/:lessonId/exercises/:exerciseId/answer

Отправка ответа пользователя на упражнение. Сервер проверяет ответ, сохраняет факт выполнения в БД (прогресс, антифарм, награды) и возвращает результат. После успешного ответа клиент запрашивает следующее упражнение через `GET /v1/lessons/:lessonId/exercises/next`.

**Заголовки запроса**

| Заголовок        | Значение |
|------------------|----------|
| `Authorization`  | `Bearer <accessToken>` |

**Параметры пути**

| Параметр    | Тип   | Описание |
|-------------|--------|----------|
| `lessonId`  | string | Идентификатор урока. |
| `exerciseId`| string | Идентификатор упражнения. |

**Тело запроса**

Зависит от типа упражнения. Общий вариант:

| Поле    | Тип    | Обязательно | Описание |
|---------|--------|-------------|----------|
| `answer`| string \| array | да | Ответ: для `sentence_builder` — строка собранного предложения или массив слов в порядке; для `multiple_choice` — идентификатор выбранного варианта. |

Пример (сбор предложения):

```json
{
  "answer": "I always drink coffee in the morning"
}
```

Пример (выбор варианта):

```json
{
  "answer": "option-id-2"
}
```

**Успешный ответ** — `200 OK`

| Поле       | Тип    | Описание |
|------------|--------|----------|
| `correct`  | boolean| Верный ли ответ. |
| `correctAnswer` | string | (при ошибке) Правильный ответ для отображения. |
| `explanation`   | string | (опционально) Краткое пояснение. |
| `reward`   | object | (опционально) Начисление зернышек и т.д. |

Пример:

```json
{
  "correct": true,
  "reward": { "seeds": 2 }
}
```

**Ошибки**

| Код | Описание |
|-----|----------|
| 400 | Неверный формат тела (нет `answer`, неверный тип). |
| 401 | Токен отсутствует или невалиден. |
| 403 | Нет доступа к уроку. |
| 404 | Урок или упражнение не найдены. |
| 500 | Внутренняя ошибка сервера. |
