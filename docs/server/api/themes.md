# API: Темы

[Docs](../../README.md) → [Server](../README.md) → [API](README.md) → **Темы**

Эндпоинты для получения списка тем и (при расширении) содержимого темы (теория, уроки). Все роуты защищённые — в заголовке передаётся **`accessToken`**: `Authorization: Bearer <accessToken>`.

---

## GET /v1/themes

Возвращает список тем. Доступен только с валидным **`accessToken`**.

**Заголовки запроса**

| Заголовок        | Значение |
|------------------|----------|
| `Authorization`  | `Bearer <accessToken>` |

**Успешный ответ** — `200 OK`

Тело — массив объектов «тема»:

| Поле         | Тип   | Описание |
|--------------|--------|----------|
| `id`         | string | Уникальный идентификатор темы. |
| `name`       | string | Название темы (например, «Простое время»). |
| `description`| string | (опционально) Краткое описание темы. |
| `order`      | number | (опционально) Порядок отображения (меньше — выше в списке). |

Пример:

```json
[
  {
    "id": "topic-present-simple",
    "name": "Простое время",
    "description": "Present Simple, Past Simple, Future Simple.",
    "order": 1
  },
  {
    "id": "topic-gerund",
    "name": "Герундий",
    "description": "Глагол с окончанием -ing в роли существительного.",
    "order": 2
  }
]
```

**Ошибки**

| Код | Описание |
|-----|----------|
| 401 | Токен отсутствует, истёк или невалиден. Тело: `{ "statusCode": 401, "message": "Unauthorized" }`. |
| 500 | Внутренняя ошибка сервера. |

При 401 клиент должен попытаться обновить сессию через `POST /v1/auth/refresh`; при неудаче — см. [auth.md](auth.md) (Mini App: экран с ботом; браузерная сессия — в планах).
