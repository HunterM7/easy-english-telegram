# Список тем

[Docs](../../README.md) → [Client](../README.md) → [Screens](README.md) → **Список тем**

В меню приложения пункт называется **«Темы»**; в API это ресурс `GET /v1/topics` (сущность `Topic`) — см. [глоссарий PRD](../../product/PRD.md).

**Когда показывается:** по нажатию пункта **«Темы»** в нижнем меню.

**Назначение:** показать все темы (например, Простое время, Герундий, Пассивный залог); по нажатию — переход к содержимому темы (теория + уроки).

---

## Структура экрана

### Шапка
- Заголовок «Темы»

### Список тем
- **Вертикальный список** карточек тем
- Каждая карточка содержит:
  - **Название**
  - **Краткое описание** (1–2 предложения)
- Иконки/иллюстрации **не используются**
- Прогресс по теме **не отображается** (см. PRD 6.3)

### Состояния карточки темы

| Состояние | Описание | Визуал |
|-----------|----------|--------|
| **Доступна** | Тема открыта для прохождения | Обычный вид, кликабельна |
| **Заблокирована (не выкуплена)** | Требуется оплата зёрнышками | Приглушённая (disabled), иконка замка или метка «Купить» |
| **Заблокирована (подписка)** | Доступна только по подписке | Приглушённая, метка «Подписка» |

*Детали блокировки и покупки — в [economy.md](../../product/economy.md).*

---

## Навигация

- **Тема (клик на доступную)** → [Содержимое темы](topic-content.md)
- **Тема (клик на заблокированную)** → всплывающий баннер с предложением оформить подписку (см. ниже)
- Нижнее меню: **Темы** (текущий экран, выделен) | иконка попугая → [Главная](home.md) | **Настройки** → [Настройки](settings.md)

---

## Баннер подписки (при клике на заблокированный контент)

Всплывающий баннер (модалка) появляется при попытке открыть заблокированную тему.

**Содержимое:**
- Заголовок: «Тема заблокирована» или «Откройте полный доступ»
- Краткое описание преимуществ подписки
- Кнопка **«Оформить подписку»** → переход к оплате
- Кнопка **«Закрыть»** или иконка крестика

---

## Состояния экрана

- **Загрузка:** скелетон списка (несколько карточек-плейсхолдеров)
- **Ошибка загрузки:** заглушка с текстом об ошибке сети и кнопкой «Обновить»

---

## Адаптивность

- **Мобильный:** карточки на всю ширину, вертикальный список
- **Планшет/десктоп:** возможно увеличение ширины карточек или добавление отступов по бокам
